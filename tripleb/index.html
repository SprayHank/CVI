tripleb.js 1.24 allows you to simulate CSS 3 support for border-radius, box-shadow and background-gradient on your webpages. It uses unobtrusive javascript to keep your code clean.

It works in most major browsers - Mozilla Firefox 1.5+, Opera 9+, Safari 3+ and IE 7+. On older browsers, it'll degrade and your visitors won't notice a thing.

About TripleB

TripleB is my first attempt to get the 3 major beautifying features of CSS 3, namely "border-radius", "box-shadow" and "background-image: gradient", to run on not natively supporting browser platforms.

Because of compatibility reasons and to get the first release done, I've built in several...
Restrictions:

  • "border-type" is always "solid"
  • "border-radius" always all four corners
  • "box-shadow" blur radius max is "32px"
  • "box-shadow" color is always "black" (IE only)
  • "gradient" direction is horizontal or vertical only
  • "gradient" size is always "100%"
  • "gradient" colors are reduced to "from" and "to" color
  • Real background images with no repeating are always centered
  • Real background images with repeating starts always at 0,0
  • Real background images with repeat-x or repeat-y are currently not supported (IE only)
  • Elements must be displayed on startup to be identified by the initialization function.
  • Elements border, shadow and background styles can't be modified after the initialization.

Features:

  • Because tripleb.js uses microformat class names to set individual parameters, it's possible to see the styles on supporting browser platforms also when javascript is out of action.
  • Elements dimensions will resize automatically on events like "onresize", "onzoom" and "onchange".
  • The custom events "onzoom" and "onchange" are done by the use of an observer (for canvas only).
  • The observers interval is configurable from 50 to 5000 ms (default 1000 ms).
  • If the element was modified by tripleb.js or not, can be queried by "if(object.modified) {... }".
  • The repainting of an element can be enforced without time delay by "cvi_3b.paint(object);".
  • The elements content can always be changed by e.g. "object.lastChild.innerHTML".

Disadvantages:

  • A visual problem is the displaying of similar looking shadows. I've done my best, but I'm really dissatisfied with the result.
  • IE's vml do not support clipping. Result is, that shadows shines through if background-color is set to transparent and the gradient uses opacities lower than 100%.
  • IE 5-6 and Safari 1-2 are currently not supported.
  • Opera is very slow, especially on huge elements.
  • Because of Opera, it's important to set border and background-colors as 6-hex-digit (e.g. #ffa508).
  • Because of Opera, it's also necessary to set border-width in pixel only.

  • The concept behind the simulation makes it necessary to split the block-level element into an outer div and an inner block-level element. So that the outer element is always a div and represents the background "style" while the inner element is of the type block-level and represents the "content". If the outer div should have settings for "padding" and/or "overflow", they must be set to the inner element to work properly.

    Example: 
    <head>
<style type="text/css">
	.outer {
		margin: 0 1em 1em 0;
		padding: 0; 
		float: left;
		display: block; 
		position: relative; 
		border: solid 2px #ffa500; 
		background-color: #ffcc00;
	}
	.inner {
		display: block; 
		position: relative; 
		padding: 1em; 
		margin: 0;
	}
	/* if you want the style to be visible on natively supporting  
	   browser platforms also when javascript is out of action */
	.bradius_10 {
		-moz-border-radius: 10px;
		-webkit-border-radius: 10px;
		-khtml-border-radius: 10px;
		border-radius: 10px;
	}
</style>
</head>
<body>
	<div class="outer tripleb bradius_10"><p class="inner">
		Lorem ipsum dolor sit amet, consectetuer adipiscing elit, 
		sed diam nonummy nibh euismod tincidunt ut laoreet dolore 
		magna aliquam erat volutpat. 
	</p></div>
</body>


TripleB always uses the inbuilt browser capabilities first. As a last solution, if the combination of features don't match the browser capabilities, it creates an additional element. This element is positioned in the outer div as the first child, before the inner element. The element consists of the type canvas or div and is always named like the outer div, followed by an underscore and the suffix "canvas". The outer div can also be queried (for tripleb.js DOM modification) by "if(object.modified) {... }". The inner element can always be accessed by "object.lastChild". If the outer div is modified and the additional element consists of the type canvas, then tripleb.js has added an observer for a custom "onzoom" event. Assuming that the inner element has been changed by "object.lastChild.innerHTML" and the elements dimensions have also been changed, the repainting of the canvas element starts after the defined interval (default 1000 ms) modifiable by the tripleb property "cvi_3b.interval=200;". You can also enforce the repainting of the canvas without time delay by "cvi_3b.paint(object);".

Setting Up

<script type="text/javascript" src="tripleb.js"></script>

Using It

To get it run just add the class="tripleb" to the outer div.

Classes

  • Initialisation class "tripleb"

  • vary the border-radius by adding bradius followed by an underscore and the desired radius in pixel:
    Border radius class "bradius_16"

  • vary the box-shadow by adding bshadow followed by an underscore and the desired color as 6-hex-digit or 3-hex-digit, followed by an underscore and the desired opacity in %, followed by an underscore and the desired x-position in pixel, followed by an underscore and the desired y-position in pixel, followed by an underscore and the desired blur-radius in pixel:
    Box shadow class "bshadow_000000_100_0_0_10" - max blur-radius is 32px, max-min x/y offset +-64px

  • vary the background-gradient by adding bgradient followed by an underscore and the desired angle as Â°, followed by an underscore and the desired from-color as 6-hex-digit or 3-hex-digit, followed by an underscore and the desired from-opacity in %, followed by an underscore and the desired to-color as 6-hex-digit or 3-hex-digit, followed by an underscore and the desired to-opacity in %:
    Background gradient class "bgradient_0_ffffff_100_ffa500_50" - angle is currently limited to 0==vertical, 90==horizontal

Syntax

Set defaults using Javascript
cvi_3b.interval = 1000; //INT 50-5000 (ms)

Get some informations
FLOAT = cvi_3b.version;
STRING = cvi_3b.released;

Initialize simulation using Javascript
cvi_3b.init();

This function has been executed automatically on page load!

Repaint element using Javascript
cvi_3b.paint(element);
Example:
var ele=document.getElementById("outer");
if(ele&&ele.modified) {cvi_3b.paint(ele);}

Why use Javascript TripleB?

  • Fast, small and quite easy to implement with a little knowledge of html and css.
  • Don't need to spend time in an image editor creating images to simulate round corners, shadows and gradients.
  • It's dead easy to change the attributes of the elements.
  • Free of charge use on non-commercial and private web sites.

In older browsers, the script degrades and your visitors won't notice a thing.
